# DataOps Control Tower — Portfolio Case

## Positioning
Complement to DataFlow: while DataFlow diagnoses a tabular dataset at a point in time, Control Tower monitors reliability over time.

## Interview talking points
- Data observability vs one-off profiling
- Freshness SLA and incident response
- Schema drift and volume anomaly signals
- Explainable quality score with severity-weighted penalties
- Product scope control (3 demo connectors in MVP)

## Demo narrative
1. Open the Control Tower dashboard
2. Read overall reliability and SLA breaches
3. Inspect quality matrix status lights
4. Walk the incident timeline and recommended actions
5. Replay recent runs and discuss next hardening steps
